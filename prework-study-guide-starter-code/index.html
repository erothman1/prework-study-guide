HTML

  • Hypertext Markup Language(HTML): handles content, structure, and meaning of each element on a page of a website.
  • HTML allows us to build a sturdy frame for the webpage.
  • HTML elements are usually composed of an opening tag and a closing tag represented with ankle brackets.
  • Tags contain everything we want to include in the element.
  • To add content to the page, we can use text elements, image elements, header elements, and list elements.
  • To divide or describe content, we can use meta-related elements, content division elements, and semantic elements.
  • Child elements are elements that are nested inside other elements.
  • The parent element holds all of the children.
  • The head element contains information, or metadata, about the webpage.
  • Elements within the head are not visible to the user.
  • Inside the head element, a title element is required and is extremely important for Search Engine Optimization(SEO).
  • We can also use style elements, base elements, link elements, meta elements, and script elements inside the head element.
  • Style elements define the CSS styles associated with a website.
  • Base elements define the base URL for a webpage.
  • Link elements connect an external resource to the HTML document.
  • Meta elements define metadata such as character set, page description, keywords, author, and viewport settings.
  • Script elements embed client-side scripts in the HTML document.
  • The body element represents the visible content shown to the user.
  • All of the elements placed inside the body are seen and interacted with by the user.
  • A body element is an example of a semantic element.
  • A semantic HTML element describes its content and role with its name.
  • The h1, h2, h3, h4, h5, h6 elements represent the level of headings a given text bloc represents.
  • The h1 element will have the largest text.
  • The p element represents a pararagph or block of text.
  • The ul element represents an unordered, or bulleted list.
  • The ol element represents an ordered, or numbered, list.
  • The li element represents list items.
  • The img element contains information about images that are displayed on the webpage and can contain various attributes.
  • The src attribute in an img element defines the location of the image.
  • The alt attrubute in an image element contains a text string that describes the appearance and functionality of the image.
  • The br element creares a line of empty space, or a line break, between two elements of content.
  • The a element (stands for anchor) creates links to the same webpage or other webpages.
  • The section elements are semantic elements that define a section in a document that contains thematic content.
  • The html opening tag represents the root or start of the HTML document.
  • Tags can contain attributes, which defines additional information.
  • Elements and tags are commonly used interchangeably, but there is a difference.
  • Elements are the entire element from opening tag to closing tag.
  • Tag refers only to what appears insde the ankle brackets.

Git

  • "git" must precede every Git statement
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout main: swtitches to main branch
  • git pull origin main: pulls down latest version of main branch
  • git add -A: adds all changes from feature branch you are working on to current working branch to commit
  • git commit -m "message": the message is the commit message that describes what the commit contains
  • git push origin branch-name: pushes changes to remote base branch
